Literature Survey

Organizations are exposed to cyber-attacks that cause significant financial weight additionally to losses such as data breaches, system collapse, privacy disclosure, reputation, violation of security policies, and loss of governance. As manufacturing automated systems are evolving rapidly new vulnerabilities would arise if security is not a primary concern in design. There are layered and decentralized security approaches but these solutions lack efficiency.

Research Gap

Industrial 4.0 transaction from traditional hardware based controllers architecture to a secure smart automation software architecture is a challenge.


When, implementing security, there are decentralized and layered approaches as solutions for the implementation and maintenance of smart manufacturing systems. It was observed that the layered and decentralized security approaches were common among the literature, but those approaches affected for the decreased efficiency of the overall smart system.


Therefore to increase the efficiency a comprehensive centralized security solution was implemented and tested for industrial automated systems.

Research Problem

Most system developers do not entirely recognize the cyber security challenges when designing, implementing or maintaining an industrial 4.0 automated system as there are many inter connected devices and technologies. Also vendors and manufactures are focused on productivity rather than security.


Challenges to overcome:

  • Collaboration between different systems
  • Centralized security management
  • Secure communication
  • Insecure data
  • Initial Cost
  • Industry 4.0 plans and strategies are lacking

Research Objectives

Main Objective

Security implementation for the potential challenges of the smart garment manufacturing system.


Final Product Objective

An automated garment manufacturing system which has been securely implemented to overcome the potential security challenges in the garment manufacturing industry according to the security standards, which is also verified for authentication and access control for the utilized IoT devices, automated centralized security configurations, security updates and intrusion detection system.


Sub Objectives

  • Risk assessment
  • Standardization and policy implementation
  • Security configurations
  • Audit security configurations
  • Centralized device configuration management
  • Generate audit reports
  • Update Management
  • Authentication and access control
  • Access log visualization
  • Report generation
  • Alert user when an anomaly occurs
  • Firewall and IDS
